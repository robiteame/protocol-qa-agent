# 通讯协议问答 Agent — 开发文档导读

> 项目代号：**ProtoQA**
> 目标：构建一个针对通讯协议类文档（如 Modbus、MQTT、CAN、私有协议规范等）的智能问答 Agent。
> 技术路线：**传统 RAG（向量检索） + PageIndex（树状推理检索）** 双引擎融合。
> 定位：本文档只提供架构、流程、模块设计与开发步骤，**所有代码由你自己编写**，以此练习 Agent 开发与 Python 语法。

## 建议学习路径

```
阶段0 环境与骨架 ──► 阶段1 配置与模型适配层 ──► 阶段2 文档上传与持久化
        │
        ▼
阶段3 解析与切块 ──► 阶段4 传统RAG链路(MVP里程碑★) ──► 阶段5 PageIndex引擎
        │
        ▼
阶段6 融合与Agent循环 ──► 阶段7 API完善/前端/评测
```

**关键心态**：
- 阶段 4 结束时你就拥有一个"能用"的问答系统（MVP），后面都是增强。卡住时随时可以回退到这条基线。
- 每个阶段都有"验收标准"，自测通过再进入下一阶段，不要并行铺开。
- 遇到不会的 Python 语法，先查 [06-python-points.md](06-python-points.md) 中对应模块的提示，再查官方文档，最后才问 AI——并且问"为什么这样写"，而不是"帮我写"。
